Introduction

To access the data, you can either use the manager or directly the Eloquent model.

If you wish to extend any class, remember to update the configuration. You can publish the configuration through the command php artisan vendor:publish

Let's first check the list of all attributes

Attributes

Name Fillable Required Unique Default Comment
id No No No / Indentify the entity
title Yes No No null
body Yes Yes No null
postable_type Yes No No null
postable_id Yes No No null
created_at No No No / Indicate the date when the record was created
updated_at No No No / Indicate the date when the record was last updated
deleted_at No No No / Indicate the date when the record was soft-deleted

Model

Base class Railken\Amethyst\Models\Post

You can extends the class like the following example in app/Models/Post

namespace App\Models;

use Railken\Amethyst\Models\Post as Model;

class Post extends Model {
    // ...
}

Remember to update the configuration with new class.

Manager

Base class Railken\Amethyst\Managers\PostManager

The manager is the main class to access and manipulate your model.

Why use the manager instead of the model? Because the manager handle all the boring stuff like validation and authorization for you. Remember that the manager return always a Result.

You can extends the class like the following example in app/Managers/PostManager

namespace App\Managers;

use Railken\Amethyst\Managers\PostManager as Manager;

class PostManager extends Manager {
    // ...
}

Remember to update the configuration with new class.

Create a new entity

First you have to define a new instance of the Manager

use Railken\Amethyst\Managers\PostManager;

$manager = new PostManager();

Now you can use the method create to create a new a new entity

$params = [
    "title" => "Maia Gleichner",
    "body" => "Temporibus consectetur voluptatem odio aliquid. Eos aspernatur et tempore dolores consectetur similique laudantium. Pariatur voluptas consequatur et sint.",
    "postable_type" => "foo",
    "postable" => [
        "name" => "Minerva McDermott",
        "description" => "Recusandae nobis unde adipisci iure voluptatem. Quod qui dignissimos consequatur maiores vel qui. Qui accusantium dolorem necessitatibus.",
        "bar" => [
            "name" => "Miss Bonnie DuBuque MD",
            "description" => "Et fuga quia dolorum quas velit et ut consectetur. Eveniet et quia sunt sed aut ratione eum. Ipsam aspernatur voluptas ex natus vel fugit. Repudiandae nesciunt ipsa eius est aut."
        ]
    ]
];

$result = $manager->create($params);

Check the result of the operation

if ($result->ok()) {
    // All ok
} else {
    // Something goes wrong
}

Retrieve the entity from the result

$entity = $result->getResource();

Result

Once you've got the result you should always check if an error has occurred, if not, retrieve the resource.

use Railken\Amethyst\Managers\PostManager;

$manager = new PostManager();

$result = $manager->create([
    "title" => "Maia Gleichner",
    "body" => "Temporibus consectetur voluptatem odio aliquid. Eos aspernatur et tempore dolores consectetur similique laudantium. Pariatur voluptas consequatur et sint.",
    "postable_type" => "foo",
    "postable" => [
        "name" => "Minerva McDermott",
        "description" => "Recusandae nobis unde adipisci iure voluptatem. Quod qui dignissimos consequatur maiores vel qui. Qui accusantium dolorem necessitatibus.",
        "bar" => [
            "name" => "Miss Bonnie DuBuque MD",
            "description" => "Et fuga quia dolorum quas velit et ut consectetur. Eveniet et quia sunt sed aut ratione eum. Ipsam aspernatur voluptas ex natus vel fugit. Repudiandae nesciunt ipsa eius est aut."
        ]
    ]
]);

if ($result->ok()) {

    $resource = $result->getResource();

} else {

    // Loop through all errors
    $result->getErrors()->map(function($error) {
        return $error->toArray();
    }))

    // Retrieve an array of all errors
    $result->getSimpleErrors();

    /* The result is something like this:

        [0] => Array
            (
                [code] => FIELD_NOT_DEFINED
                [label] => field
                [message] => The field is required
                [value] =>
            )
    */

}

    // ...
}

Remember to update the configuration with new class.

Repository

Base class Railken\Amethyst\Repositories\PostRepository

The repository is the class used to perform queries.

use Railken\Amethyst\Managers\PostManager;

$manager = new PostManager();

$repository = $manager->getRepository();

Retrieving an entity

$repository->findOneBy(['id' => 1]);
$repository->findOneById(1);

Retrieving all entities

$repository->findAll();

Performing a query using \Illuminate\DataBase\Eloquent\Builder

$repository->newQuery()->where('id', 1)->first();

You can extends the class like the following example in app/Repositories/PostRepository

namespace App\Repositories;

use Railken\Amethyst\Repositories\PostRepository as Repository;

class PostRepository extends Repository {
    // ...
}

Remember to update the configuration with new class.

Serializer

Base class Railken\Amethyst\Serializers\PostSerializer

The serializer is used to serialize an entity, you can retrieve it from the data.

use Railken\Amethyst\Managers\PostManager;

$manager = new PostManager();

$serializer = $manager->getSerializer();

And use it so serialize an entity Retrieving an entity

$entity = $repository->findOneById(1);
$serializer->serialize($entity)->toArray(); // Returns an array

You can extends the class like the following example in app/Serializers/PostSerializer

namespace App\Serializers;

use Railken\Amethyst\Serializers\PostSerializer as Serializer;

class PostSerializer extends Serializer {
    // ...
}

Remember to update the configuration with new class.
